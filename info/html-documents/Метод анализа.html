Метод анализа

Алгоритм рекурсивного спуска для анализа логических выражений в грамматике языка C++ G[‹Expr›]:

  1. Начинаем с символа ‹Expr›.
  2. Метод для символа ‹Expr› вызывает метод для символа ‹AndExpr›.
  3. Метод для символа ‹AndExpr› вызывает метод для символа ‹NotExpr›.
  4. Метод для символа ‹NotExpr› проверяет наличие оператора отрицания '!', если он есть, то вызывает метод для символа ‹SimpleExpr›, иначе продолжает непосредственно с метода для символа ‹SimpleExpr›.
  5. Метод для символа ‹SimpleExpr› проверяет тип текущего токена. Если это идентификатор, логический литерал или число, то переходим к следующему токену. Если это символ '(', вызываем метод для символа ‹Expr›, затем проверяем наличие символа ')', иначе возвращаем ошибку.
  6. Метод для символа ‹SimpleExpr› проверяет, является ли текущий токен оператором сравнения. Если это так, вызываем метод для символа ‹ComparisonExpr›.
  7. Метод для символа ‹ComparisonExpr› вызывает метод для символа ‹ArithmeticExpr›, затем проверяет наличие оператора отношения и вызывает метод для символа ‹ArithmeticExpr› еще раз.
  8. Метод для символа ‹ArithmeticExpr› вызывает метод для символа ‹Term› и проверяет наличие операторов сложения или вычитания, вызывая метод для символа ‹Term› при их наличии.
  9. Метод для символа ‹Term› вызывает метод для символа ‹Factor› и проверяет наличие операторов умножения, деления или остатка от деления, вызывая метод для символа ‹Factor› при их наличии.
  10. Метод для символа ‹Factor› проверяет тип текущего токена. Если это идентификатор, целочисленное или вещественное число, то переходим к следующему токену. Если это символ '(', вызываем метод для символа ‹ArithmeticExpr›, затем проверяем наличие символа ')', иначе возвращаем ошибку.
  11. Если текущий токен не является допустимым для данного выражения, возвращаем ошибку.
  12. Если достигнут конец выражения и ошибок не возникло, возвращаем успешный результат.
